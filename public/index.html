100 apps · researched by an agent · accuracy-verified

Which of these 100 apps can be an agent toolkit today — and which need a sales call?

An agent (Claude + Composio's SDK & managed toolkits + Firecrawl) researched every app's auth, self-serve path, API surface and existing MCP, then verified its own answers against live docs and a hand-audited sample.

The headline

What the 100 apps tell us

  • OAuth2 is the dominant auth method (71 of 100 apps) — a managed-auth layer covers most of the field.
  • 76/100 apps are self-serve — the easy wins; the rest need paid plans or outreach.
  • 72/100 apps are buildable into an agent toolkit today; 26 come with caveats and 2 are blocked outright.
  • 70/100 apps are instant wins — self-serve credentials AND buildable now — while 14 need a paid plan, admin approval, or a sales conversation first.

Honesty

Limitations & the apps that fought back

Two things we'd flag to a reviewer before trusting this at scale: (1) MCP over-detection — the agent marks existing_mcp: yes far too readily (97/100), because it conflates "has an API you could wrap" with "ships a real MCP server"; our audit puts this field at ~60% (on a 5-app slice). (2) Determinism — a transient scrape/LLM error used to silently drop an app to its first-pass answer; the pipeline now logs every fallback, and the committed cache makes the reported run byte-reproducible.
